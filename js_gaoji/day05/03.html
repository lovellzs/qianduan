第五天笔记

构造函数的问题

构造函数本身是一个函数，在调用的时候有多种调用方式。

① new 构造函数（）调用

② 构造函数（）调用

以上，第二种调用方式存在作用域安全的问题

作用域安全的构造函数

<script>

    //01 提供一个构造函数
    function Person(name) {

        //容错处理
        if (this instanceof Person)
        {
            //设置实例属性和方法
            this.name = name;
            this.showName = function () {
                console.log(this.name);
            }
        }else
        {
            return new Person(name);
        }

    }

    //02  创建对象
    var p1 = new Person("zhangsan");
    var p2 = new Person("lisi");
    p1.showName();
    p2.showName();

</script>

新的问题

说明:上面的代码在使用借用构造函数方式继承的时候存在问题，无法创建出的对象，原因在于通过call或者是apply函数调用的时候，内部的this绑定的子类型中的对象，因此在使用instanceOf 判断的时候，结果为假

解决：可以设置让子类型（构造函数）的原型对象为父类型的一个实例。

代码示例

<script>

    //01 提供一个构造函数
    function Person(name) {

        //容错处理
        if (this instanceof Person)
        {
            //设置实例属性和方法
            this.name = name;
            this.showName = function () {
                console.log(this.name);
            }
        }else
        {
            return new Person(name);
        }

    }

    //02  创建对象
    var p1 = new Person("zhangsan");
    var p2 = new Person("lisi");
    p1.showName();
    p2.showName();

    // 使用借用构造函数来实现继承
    function Boy(name){
        Person.call(this,name);
    }

    //需要结合原型继承来解决 instanceOf 实例对象检测的问题
    Boy.prototype = new Person();

    var boy = new Boy("测试的名字");
    boy.showName();

</script>
